# AI漫画生成器

## 功能概述

将故事转化为漫画，通过AI生图技术生成多张连贯的单图漫画，保持人物一致性，最终组装成可阅读的HTML漫画页面。

---

## 输入输出

### 输入
| 字段 | 说明 | 示例 |
|------|------|------|
| 故事内容/剧本 | 用户提供的故事或主题 | "职场逆袭"、"创业励志"、"搞笑反转"（用户输入则跳过创作） |
| 角色设定 | 主角外观描述 | "25岁男性，戴眼镜，穿西装"（可选） |
| 篇幅要求 | 几张图 | "10张左右"或"3-5张短篇" |
| 风格要求 | 画面风格 | "温暖治愈"、"搞笑"、"写实" |

### 输出
| 内容 | 说明 |
|------|------|
| 分镜剧本 | 表格形式，包含：序号、场景、画面描述、人物动作、对话/旁白 |
| 漫画图片 | 每张图独立文件 (panel01.jpg...) |
| HTML阅读器 | 可翻页的漫画网页，文字叠加在图片上 |
| ZIP包 | 打包下载 |

## 工作流程

### Step 1: 创作剧本（仅当用户未提供内容时执行）

**灵活篇幅策略：**
- 短篇（3-5张）：聚焦1个核心冲突，快速推进
- 中篇（6-8张）：完整起承转合
- 长篇（10+张）：深度铺垫 + 多次反转

**剧本模板：**

```
| 张 | 场景 | 画面描述 | 人物动作 | 对话/旁白 |
|:--:|------|----------|----------|-----------|
| 1 | [时间+地点] | [完整场景] | [具体动作] | [完整对话] |
```

**剧本要求：**
- 每张图必须有完整的画面描述
- 每张图必须有具体的人物动作
- 每张图必须有完整的对话或旁白
- 不能只写"领导骂人"，要写"领导眉头紧皱，手指点着屏幕，俯视小王"

**反转漫画结构（长篇10张）：**
- 第1-3张：铺垫困境（压抑、被欺负）
- 第4-6张：意外转折（希望出现、反转开始）
- 第7-10张：高潮结局（强烈反转、有冲击力）

**短篇结构（3-5张）：**
- 第1张：开场冲突
- 第2-3张：推进 + 反转
- 最后张：神转折结局

### Step 2: 定义人物（关键！）

**每张图必须使用完全相同的人物描述！**

```
主角"小王"固定人设（每次生图必须用这段）：
25岁中国男性，圆脸，黑色短发，戴黑框眼镜，
穿深蓝色西装外套+白色T恤，表情[根据剧情变化]
```

**配角描述示例：**
```
领导：35-40岁男性，秃顶，穿深色西装，表情凶狠
实习生：22岁女性，扎马尾辫，穿白衬衫，笑容甜美
```

### Step 3: 生成图片（关键！）

**重要：必须是单图！不要宫格！不要分格！**

**提示词公式：**
```
一张[风格]漫画，画面是[完整场景描述]，
[人物完整外观]+[具体动作]+[表情]，
[背景细节]，[色调]，[质感]
```

**错误示例：**
```
❌ "三宫格漫画"
❌ "上下两格"
❌ "四个分格"
```

**正确示例：**
```
一张温暖治愈风漫画，画面是现代办公室场景，
一个25岁中国男性，圆脸，黑色短发，戴黑框眼镜，
穿深蓝色西装外套+白色T恤，
低着头眉头紧锁，表情沮丧委屈，双手紧握被撕碎的方案，
旁边站着35岁秃顶男性领导，手指戳着屏幕，表情严厉，
办公室背景，白领工位，落地窗，柔和自然光，
暖黄色调，手绘插画质感，故事感强
```

**每张图的提示词结构：**
```
主体（固定人设）+ 具体动作 + 表情 + 场景 + 配角（如果有）+ 色调 + 质感
```

### Step 4: 组装HTML漫画站

从 `assets/comic-reader/` 复制模板文件：

```
output/
├── index.html
├── jszip.min.js
└── images/
    ├── panel01.jpg
    ├── panel02.jpg
    └── ...
```

**HTML模板特点：**
- 文字叠加在图片底部（对话框样式）
- 支持键盘左右键翻页
- 右上角一键打包下载

### Step 5: 打包下载

用 zip 打包所有文件，发送给用户。

## 注意事项

1. **图片必须是单图** - 不能用宫格、不能分格，每张图就是一个完整画面
2. **人物一致性** - 每张图的主角必须长得完全一样（用固定描述）
3. **剧本要完整** - 每张图必须有画面+动作+对话/旁白
4. **反转要强烈** - 结局要有冲击力
5. **灵活篇幅** - 根据用户需求，短篇/中篇/长篇皆可
6. **智能跳过** - 用户提供内容时，直接从生图开始
